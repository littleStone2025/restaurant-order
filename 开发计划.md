## 1. 项目概述

### 1.1 项目目标

搭建一套面向堂食顾客的轻量级网页自助点单系统，支持扫码即用、在线下单、微信支付，覆盖顾客点餐全流程，降低餐厅人力成本，提升翻台率。

### 1.2 核心功能范围

| 模块 | 功能点 |
|------|--------|
| 菜单浏览 | 分类导航、菜品卡片、角标（热销/新品/推荐） |
| 菜品搜索 | 关键词实时过滤 |
| 菜品详情 | 图片/描述/标签弹窗 |
| 购物车 | 增减/清空/实时计价、服务费自动计算 |
| 下单结算 | 备注填写、订单提交 |
| **微信支付** | JSAPI 支付、支付结果回调 |
| 订单状态 | 下单成功弹窗、订单号/金额展示 |
| 后台管理（管理端） | 菜品增删改查、上下架、订单列表 |

### 1.3 项目边界

**纳入范围：**
- 顾客点单前台（Web H5，含微信内嵌浏览器适配）
- 商家后台（菜品管理、订单查看）
- 微信支付对接

**不纳入范围（本版本）：**
- 会员系统、积分体系
- 数据大屏、营业统计
- 多门店支持
- APP 原生开发

### 1.4 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端 | HTML5 + CSS3 + Vanilla JS | 零框架依赖，包体 < 150KB |
| 后端 | Node.js + Express（或 Fastify） | 轻量 RESTful API |
| 数据库 | MySQL 8.0 | 菜品表、订单表、分类表 |
| 缓存 | Redis | 库存锁、接口限流、购物车会话 |
| 微信 SDK | JSSDK + JSAPI | 微信内置浏览器鉴权 + 支付 |
| 支付网关 | 微信支付 V3 API | JSAPI 支付（需商户号） |
| 部署 | Docker + Nginx + CDN | 容器化 + 静态资源加速 |
| 域名 | 需已备案域名 | 微信支付必须条件 |

### Phase 1 — 后端基础建设（第 2-3 周）

**任务清单：**

1. **数据库初始化**
   - 建库建表脚本（categories / dishes / orders / order_items）
   - 初始化种子数据（7个分类、30道菜品）
   - Redis 连接配置

2. **菜品与分类接口**
   - `GET /api/categories` — 获取全部分类 + 菜品数
   - `GET /api/dishes?category_id=&available=1` — 菜品列表（支持分类过滤）
   - `GET /api/dishes/:id` — 菜品详情

3. **基础中间件**
   - 日志中间件（Morgan / Winston）
   - 错误处理中间件
   - CORS 配置
   - 接口限流（Redis + sliding window，60次/分钟/IP）

4. **单元测试**
   - 菜品接口测试用例 ≥ 15 条

#### Week 3：订单 API

**任务清单：**

1. **订单接口**
   - `POST /api/orders` — 创建订单（原子事务，生成 order_id）
   - `GET /api/orders/:orderId` — 查询订单状态
   - `PATCH /api/orders/:orderId/status` — 更新状态（后厨操作）

2. **微信 JSSDK 签名接口**
   - `GET /api/wx/jssdk-config` — 返回 wx.config 所需签名参数
   - 生成签名算法（noncestr + timestamp + url + jsapi_ticket）

3. **微信统一下单接口集成**
   - `POST /api/pay/create` — 调用微信 `https://api.mch.weixin.qq.com/v3/pay/transactions/jsapi`
   - 生成 `prepay_id` 并组装支付参数
   - 签名生成（RSA + API v3）

4. **支付回调接口**
   - `POST /api/pay/callback` — 接收微信支付结果通知
   - 回调验签（平台证书）
   - 订单状态更新（`pay_status: paid`）

### Phase 3 — 微信支付集成（第 4-5 周）

#### Week 4：微信 JSSDK 接入

**前置条件：** 微信商户号已申请、H5 域名已在微信公众平台授权

**任务清单：**

1. **微信公众号配置**
   - JS接口安全域名设置
   - 网页授权域名设置
   - 获取 `AppID` 和 `AppSecret`

2. **前端 JSSDK 初始化**
   - 引入微信 JSSDK（`https://res.wx.qq.com/open/js/jweixin-1.6.0.js`）
   - 调用 `/api/wx/jssdk-config` 获取签名
   - `wx.config({ debug: false, jsApiList: ['chooseWXPay'] })`
   - 静默授权获取 `openid`（code 换 openid）

3. **openid 获取流程**
   ```
   用户打开页面 → 检测微信内 → redirect 微信授权页 → 拿 code
   → /api/wx/code2session → 获取 openid → 存入 sessionStorage
   ```

#### Week 5：JSAPI 支付全流程

**任务清单：**

1. **后端统一下单**
   ```
   POST /api/pay/create
   Body: { orderId, openid, totalFee(分), description, notifyUrl }
   → 调用微信统一下单 API → 返回 { prepay_id }
   → 前端返回支付参数 { appId, timeStamp, nonceStr, package, signType, paySign }
   ```

2. **前端唤起支付**
   ```javascript
   wx.chooseWXPay({
     timestamp, nonceStr, package, signType, paySign,
     success: () => { // 支付成功 → 查询订单状态 → 展示成功页 },
     fail: (err) => { // 支付失败 / 取消 → 展示失败原因 }
   });
   ```

3. **支付结果处理**
   - 前端成功回调 → 跳转成功页（动画 + 订单摘要）
   - **关键**：最终以服务端回调结果为准，禁止前端直接标记已支付
   - 后端回调失败 → 告警 + 人工核查

4. **支付安全加固**
   - 所有金额单位统一为**分**（后端计算，前端只传 `orderId`）
   - `paySign` 使用 RSA SHA256 签名（API v3）
   - 回调验签：使用微信平台证书验签（API v3）
   - 订单状态机：`(pending) → (paid) → (cooking) → (done)`

**Phase 3 交付物：**
- [x] 微信 JSSDK 鉴权流程（静默授权）

- [x] 支付回调验签服务

## 4. 微信支付集成方案

### 4.1 支付模式选择

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **JSAPI 支付** | 通过 openid 在微信内唤起支付 | **首选**，体验最优 |
| H5 支付 | 外部浏览器跳转微信支付 | 降级方案 |
| Native 支付 | 生成二维码 | PC 端点餐机 |

**本项目选用：JSAPI 支付**

### 4.3 关键 API 清单

| 接口 | 微信 API | 说明 |
|------|---------|------|
| 获取 openid | 公众号授权 `sns/jscode2session` | 静默授权，无需用户确认 |
| 获取 jsapi_ticket | `cgi-bin/get_jsapi_ticket` | 需全局缓存 7200 秒 |
| JSSDK 签名 | 自实现 | `jsapi_ticket` + `noncestr` + `timestamp` + `url` |
| 统一下单 | `POST /v3/pay/transactions/jsapi` | 生成 `prepay_id` |
| 支付结果通知 | 微信主动推送 | POST 至我方回调地址 |
| 查询订单 | `GET /v3/pay/transactions/id/{transaction_id}` | 主动查询补充 |

### 4.4 微信支付申请材料清单

| 材料 | 说明 | 难度 |
|------|------|:----:|
